# 术语与八镜框架

## 输出规则

每个围绕行星或主题的小节，都按这个顺序组织：

1. `现实判断`：现实里会怎么表现
2. `盘面抓手`：哪些盘面事实支撑这个判断
3. `使用提醒`：该怎么用，或者哪里要保留弹性

不要只输出术语、表格或未解释的缩写。

## 回答质量闸门

下面这些不是建议，是最低门槛。

### 1. 每节都要先说人话

每个主要小节开头先用 2 到 4 句普通话解释：

- 这在现实里会怎么表现
- 用户最该先抓住什么
- 最主要的阻力或代价在哪里

不要一上来就堆表格、缩写、镜面名词或英文标签。

### 2. 每节都要有证据锚点

每个主要小节至少显性写出两类证据，且不能都只是抽象术语。优先从下面选：

- 宫位与宫主
- 行星状态
- dasha
- 命主盘（Rashi chart）
- 九分盘（Navamsha chart）
- SAV、BAV、Shadbala
- karaka
- yoga

如果只有一类证据，就把结论降级，不要写成定论。

### 3. 每节都要交代可用度

每个主要小节末尾至少交代一项：

- 这条结论为什么相对稳定
- 这条结论受哪些缺数据影响
- 用户实际使用时该把它当趋势、窗口，还是高确定性判断

### 4. 时间判断必须能落地

只要提到时间窗口，就必须同时回答三件事：

- 窗口落在哪个具体区间
- 触发逻辑是什么
- 这个窗口适合推进什么，不适合浪费在哪

### 5. 禁止松答模板

避免下面这些写法：

- 只有形容词，没有盘面依据
- 只有术语，没有现实翻译
- 只有趋势，没有时间
- 只有优点，没有代价
- 只有结论，没有不确定性

## 事件回看口径

既往事件回看现在使用**双层口径**：面向用户的总结标签 + 面向系统校准的诊断维度。

### 旧命中等级（保留作为用户侧总结标签）

- `高贴合`：时间窗和事件性质都能形成清楚对应
- `有限贴合`：时间大致合适，但事件类型更宽、更弱，或只对上一半
- `失配`：时间或事件性质都没有形成可靠对应

不要把 `有限贴合` 涂改成 `高贴合`，也不要把 `失配` 硬解释成"只是表现形式不同"。

### 诊断维度（每次回看必须输出，用于系统校准）

在旧标签之外，每条既往事件必须额外输出以下三个维度：

| 维度 | 含义 | 评估 |
|------|------|------|
| **信号命中** | 星盘在该时间段是否确实有对应类型的结构信号激活？ | ✅ 命中 / ⚠️ 部分命中 / ❌ 未命中 |
| **推断合理性** | 给定当时的星盘信号 + 背景事实，从信号到具体推断的逻辑链是否成立？ | ✅ 合理 / ⚠️ 方向对但精度偏 / ❌ 逻辑有缺陷 |
| **信息完备度** | 推断时所掌握的背景事实是否足够支撑这个精度的结论？ | 🔴 高 / 🟡 中 / ⚪ 低 |

三个维度的诊断价值详见 SKILL.md 分层推断规则层 3 的「诊断矩阵」。

## 分层推断术语

### 背景事实卡

分析前按需收集的结构化用户信息。仅收集当前分析方向所需的领域：

| 领域 | 收集项 |
|------|--------|
| 通用 | 性别、年龄区间、出生时间精度 |
| 感情 | 当前状态（已婚/恋爱/单身/离异）、关键关系稳定性 |
| 事业 | 当前阶段（学生/在职/创业/待业/退休）、行业方向 |
| 迁移 | 当前居住地、异地关系强弱、搬迁动力 |
| 健康 | 已知重大健康问题 |

**事实与自述的边界**：可被外部验证的客观信息（婚姻状态、职业阶段、居住城市）是事实，可进入背景事实卡。主观判断（「我性格内向」「我运气不好」）是自述，不能进入，需要被盘面验证。

### 层 0 / 层 1 / 层 2 / 层 3

分层推断的四个独立分析阶段：

- **层 0（星盘结构信号）**：从命盘数据中提取的结构信号——Dasha 周期、宫位激活、行星状态、Yoga。输出信号类型、强度和复数可能方向。禁止反向推导和经历=天赋。
- **层 1（背景事实卡）**：用户已知的客观背景信息，用于缩小层 0 的复数可能方向。不用于否定层 0 信号本身。
- **层 2（联合推断）**：信号 × 背景 → 具体结论。每个推断必须透明展示推理链、纯星盘版本、边界条件和反事实检查。
- **层 3（事后校验）**：通过信号命中、推断合理性和信息完备度三个维度诊断推断链，回答「如果不准，断在哪里」而非简单判对错。

## 细节层级护栏

结论不要混层。先判断自己现在说的是哪一层：

- `结构层`：例如技术型、大平台、离家发展、工作与学习并行
- `窗口层`：例如某一段时间职业定型，或某一段时间承压并转场
- `专名层`：例如某家公司、某个国家、某个具体病名、某一个领导

推荐做法：

- 结构层可以作为高置信度主结论
- 窗口层可以结合 dasha 做中高置信度判断
- 专名层只能在用户已经给出现实事实时作为对照，不能假装是盘面原生可读的专有答案

特别注意：

- “技术型工作”是结构层
- “进入大型平台并出现收入抬升”是窗口层
- “某一家具体公司”是专名层
- “身体吃不消、筋骨压力上升”是结构或窗口层
- “某一个具体医学诊断”是专名层

## 精确计算规则

八镜框架负责解释结构，不负责替代精确算术。

只要结论依赖下面这些项目：

- 度数换算
- SAV、BAV 总和
- Shadbala 排序
- dasha 日期间隔
- 明确的通过、失败或跳过判断

就先从原始数据或本地工具拿到数值，再写解释，不要边写边心算。

## 八镜定义

| 观察镜面 | 它在看什么 | 它如何影响结论 |
|---|---|---|
| 主题归属 | 这颗星或这个宫位到底在负责哪类人生主题 | 先决定“它管什么”，再谈“它做得怎样” |
| 运行状态 | 燃烧、逆行、稳定度和损耗感 | 决定推进方式是顺、迟、绕，还是内耗 |
| 资源水位 | 它手里有没有足够资源把事情做出来 | 决定主题能否持续，不只是偶尔闪一下 |
| 落点环境 | 这股力量落在轻松区、磨损区，还是高压区 | 决定现实成本和摩擦位置 |
| 兑现通道 | 结果是怎么落地的，是直接、绕路还是先堵后通 | 决定成果出现的方式 |
| 外力牵引 | 谁在帮助、压制、改道或放大它 | 决定外部环境如何参与这件事 |
| 熟成节律 | 它更偏早显、晚显，还是需要时间酝酿 | 帮助解释为什么某个主题在人生不同阶段手感不同 |
| 格局放大 | yogas、组合和系统级加权 | 决定哪些信号会被整体放大或修饰 |

## 运行状态子维度

"运行状态"不是一句话能交代的模糊标签。以下子维度必须在分析中显性检查，不能只挑最方便的一两项说。

### 燃烧（Combustion）

行星距太阳过近时，能量被日光"覆盖"，表现力打折。每颗行星的燃烧临界不同：

| 行星 | 燃烧临界（距日度数） | 逆行时调整 |
|------|---------------------|-----------|
| 月亮 | 12° | — |
| 火星 | 17° | — |
| 水星 | 14° | 逆行时 12° |
| 木星 | 11° | — |
| 金星 | 10° | 逆行时 8° |
| 土星 | 15° | — |

- Cazimi（距日 < 1°）：仍属燃烧，但精确合日的行星可能获得一种"聚焦"效应
- 燃烧 ≠ 行星作废，而是"需要借他人的舞台才能表现"——燃烧行星管的事往往需要依托别人（上司、平台、伴侣）才能发光

### 逆行修正（基于 KN Rao 体系）

逆行不只是"能量内收"，它对不同状态的行星有不同的翻转效果：

- **落陷 + 逆行**：视同入旺（Neecha Bhanga 的一种形式）。最坏的处境被逆行的反思力翻转
- **入旺 + 逆行**：视同落陷。最强的位置被逆行拉回——有能力但表达方式不顺畅
- **入庙 + 逆行**：品质仍在，但输出方式是"向内深钻"而非"向外自然释放"
- 逆行行星管的事：兑现节奏比顺行慢，但持久度常常更高

### 行星年龄状态（Baladi Avasthas）

行星在星座内的度数决定了它处于哪个"人生阶段"，影响能量输出模式：

| 年龄阶段 | 典型度数区间 | 能量特征 |
|----------|-------------|---------|
| 婴儿（Baala） | 0°–6° | 潜力未激活，需要外部触发和培养 |
| 少年（Kumara） | 6°–12° | 有冲劲但毛躁，适合学习期 |
| 青年（Yuva） | 12°–18° | 主动输出，执行力最强 |
| 壮年（Vriddha） | 18°–24° | 经验丰富但体力下降，靠策略而非蛮力 |
| 老年/寂灭（Mrita） | 24°–30° | 自动执行模式——不再主动发起，但一旦被调动就精准 |

- 婴儿和老年的行星：不适合"主动出击"式策略，更适合"等风来"
- 青年行星：最直接的行动力来源，但可能缺乏耐心

### 情绪与活动状态（Mood & Activity Avasthas）

由 JHora 根据复杂规则计算。如果原始数据中有，应一并纳入判断：

**情绪状态**常见取值：
- Saanta（平静）：稳定输出，不耗神
- Mudita（喜悦）：积极正向，事情容易推动
- Khala（顽皮）：容易在规则边界试探，有创意但不够安分
- Duhkhita（痛苦）：内耗大，需要特别关注这颗星管的领域
- Deena（沮丧）：能量低迷，管的事容易停滞

**活动状态**常见取值：
- Prakasana（发光中）：主动对外辐射，做的事情容易被看见
- Sayana（躺卧）：被动状态，需要外界推动才会动
- Gachchhati（行走中）：正常运转
- Yuddha（战斗中）：处于竞争/冲突状态

### 使用要求

- 至少显性检查燃烧和逆行状态，这是"运行状态"镜面的最低交付
- 如果 JHora 数据中包含年龄/情绪/活动状态，必须纳入分析，不可忽略
- 如果某颗行星多项子维度叠加（如"落陷 + 逆行 + Mrita 年龄 + Sayana 活动"），要明确说出叠加效果，不能只说单项

## 推荐用法

| 推荐表述 | 大白话方向 |
|---|---|
| 主题归属很核心 | 说这颗星和身份、方向、自我定义高度相关 |
| 资源水位充足 | 说它不仅有想法，也有条件把事做下去 |
| 落点环境磨损大 | 说事情能做，但过程贵、慢或摩擦多 |
| 兑现通道绕路 | 说结果不是没有，只是常常先曲折后成形 |
| 外力牵引偏强 | 说这件事很容易被环境、人际或制度放大 |
| 熟成节律偏晚 | 说它不是早年主题，后期更容易看见成形 |

## 裁定顺序

当信号互相打架时，按下面的优先级：

1. 先定 `主题归属`
2. 再看 `运行状态`
3. 用 `资源水位` 和 `落点环境` 判断成本
4. 用 `兑现通道` 判断结果怎么出来
5. 用 `外力牵引` 与 `格局放大` 做修正
6. 用 `熟成节律` 解释阶段差异

## 翻译指导

- 先翻镜面，再给现实话
- 不要把私人造词包装成标准术语
- 只在真正有帮助时才显性写专业词
- 尽量把盘面逻辑改写成一句现实里听得懂的话

推荐：

- `这条事业线不是没潜力，而是落点环境偏磨损，所以更像先累积方法和口碑，再慢慢放大。`

避免：

- 只堆英文缩写和标签

## SAV/BAV 量化规范

Ashtakavarga 是 Parashari 体系中最核心的量化工具。它不是某个 skill 的"私有参数"，而是衡量宫位级别强弱的公共标尺。

### 宇宙常数（强制校验项）

只要原始资料包含 SAV/BAV 数据，以下校验必须在一致性检查阶段完成：

**SAV 总和**：12 个宫的 SAV 值之和恒等于 **337**。不通过 → 数据读取有误，逐宫核实。

**BAV 行常量**：每颗行星在 12 个宫的 BAV 分值之和是固定常数：

| 行星 | Sun | Moon | Mars | Mercury | Jupiter | Venus | Saturn |
|------|-----|------|------|---------|---------|-------|--------|
| BAV 行和 | 48 | 49 | 39 | 54 | 56 | 52 | 39 |

**BAV 列→SAV 交叉验证**：每个宫位的 7 颗星 BAV 之和 = 该宫 SAV 值。不通过 → 行校验 + 列校验交叉定位错误点。

### 宫位 SAV 阈值分档

拿到 SAV 数据后，按以下分档判断宫位的"硬件带宽"：

| SAV 值 | 带宽评级 | 现实含义 |
|--------|---------|---------|
| ≥32 | 充裕 | 事情自动推进，投入产出比高 |
| 28–31 | 良好 | 正常努力即有回报 |
| 24–27 | 中等 | 需要策略和持续投入 |
| 20–23 | 偏紧 | 付出多回报少，容易焦虑 |
| <20 | 匮乏 | 该领域天然吃力，需要外部补给或迂回策略 |

### 掌宫资源评估

要判断一颗行星"手里有多少货"，把它掌管的两宫的 SAV 相加：

- ≥60：强掌宫组合——这颗星有充足的资源完成职责
- 50–59：中等——够用但不宽裕
- <50：偏弱——它的职责需要其他行星帮衬

如果是单宫掌管（如 Rahu/Ketu），只用单宫 SAV 评估，但置信度降半档。

### 落宫 SAV 与 BAV 配比

行星落在哪个宫，那个宫的 SAV 就是它的"路况"。但还要看这颗星自己的 BAV 在该宫的贡献——如果某宫 SAV 高但主要是别的星贡献的、这颗星自己的 BAV 很低，意味着"路好走但你得搭别人的车"。

### 缺 SAV/BAV 时的降级策略

这是 zh 目前最关键的缺口。当 SAV/BAV 无法校验时：

1. 一致性检查回执中明确写 "SAV/BAV ⏭️ 跳过，宫位级量化判断降级"
2. 宫位强弱比较只做定性判断（基于宫主星状态、宫内星有无、相位），不做"X 宫比 Y 宫强"级别的精确排序
3. "资源水位"和"落点环境"两个镜面改用 Shadbala 和 Vimsopaka 作为二级替代数据源
4. 在报告的"落地提醒"中声明：SAV/BAV 缺失导致宫位级精细判断置信度为 🟠 中

**与八镜框架的对接**：
- "资源水位"镜面 → 优先取掌宫 SAV 和 Shadbala 两项数据
- "落点环境"镜面 → 优先取落宫 SAV 分档，辅以宫位性质（角宫/三分宫/凶宫）
- 两镜面中至少一个需要 SAV 数据；若都没有 → 两镜面均降级

## 角宫主效应（Kendradhipati Dosha）

角宫（1/4/7/10）主星有一个特殊性：吉星管角宫时，其天然吉性会被角宫的"世俗事务压力"削弱——不是变凶，而是"偷懒"。凶星管角宫时，其天然凶性反而被角宫的"责任感"约束——不是变吉，而是"自律"。

这意味着：

- 金星管 7H（角宫）：婚姻/合作资源有，但需要主动争取，不会自动送上门
- 土星管 10H（角宫）：事业压力可控，土星的压制力被转化为结构化的努力
- 月亮管 4H（角宫）：内心安全感需要外在成就来确认，而非自然拥有

在分析角宫主星时，必须检查这一效应。它解释了为什么有些"看起来很好的配置"在实际生活中需要额外用力。

## 缺数据时的标准写法

如果原始资料不完整，不要硬补。优先用下面这个顺序表达：

1. `已知`：这次能稳定读取到什么
2. `未知`：缺了什么关键字段
3. `影响`：这会让哪些结论降级
4. `暂定判断`：在现有资料下还能说到哪里

## 反过拟合提醒

- 不要把用户后来补充的公司名、城市名、专业名、病名，倒灌回盘面，再说成“原来一开始就能看出来”
- 不要把“权威冲突”自动翻译成“某一个人故意害你”
- 不要把“高压与耗损”自动翻译成单一医学诊断

## 九分盘的定位

九分盘（Navamsha chart）不是第二张独立命盘，它更像对主结构的复核层。

它主要回答五件事：

- 主盘里的承诺成熟后还站不站得住（内核品质）
- 兑现时要付出什么代价（变现阻力）
- 最终果实更容易落在哪类主题上（果实投射）
- 承诺的"担保人"是否可靠（房东审计）
- 成果放在哪个"仓库"是安全的还是一碰就碎的（安全性）

九分盘负责看成熟后的质量、耐久度和承载环境，不负责抹掉命主盘的原始结构。

### D9 复核五维度

每颗关键行星的 D9 复核必须至少覆盖以下五个维度中的四个：

**1. 内核品质**：行星在 D9 的落座相对于 D1 是升级还是降级？
- Vargottama（D1 和 D9 同星座）：最高稳性——承诺 150% 可兑现
- 入旺/入庙：100% 兑现
- 友方星座：正常兑现
- 落陷：承诺可能违约，需检查 Pushkara 补丁（特定度数范围内的落陷会被修复）

**2. 安全性**：D9 落宫属于哪类"仓储环境"？
- 金库区域（1/2/4/5/7/9/10/11 宫）：成果稳定，不易流失
- 摩擦区域（3/6 宫）：兑现需要竞争和主动"卡位"
- 有毒区域（8/12 宫）：成果可能得而复失，或兑现过程伴随消耗
  - 严重度排序：D9 落 8 宫 > 12 宫 > 落陷 > 6 宫

**3. 房东审计（Dispositor 链条）**：D9 落座的守护星状态如何？
- 守护星（Dispositor）在 D9 中健康 → "支票可以兑现"
- 守护星在 D9 中落陷/受损 → "支票可能跳票"——这颗星承诺的东西，担保人自己都不够强
- 守护星和主盘大运主重合 → 兑现窗口和主运共振

**4. 变现阻力**：D1 星座 → D9 星座的位移大小。
- 位移 = |D9 星座序号 − D1 星座序号|（以 12 为模）
- 位移在 6/8/12 区间 → 变现需要"绕路"，过程有过敏反应
- 位移 ≤ 3 → 直接兑现

**5. 果实投射**：D9 落座对应回 D1 哪个宫位？
- 把 D9 星座映射到 D1 的对应宫位（两者同星座即可对应）
- 这颗行星最终产出的果实，会掉在 D1 的那个宫位代表的领域
- 这是连接"深层品质"和"现实结果"的关键桥梁
